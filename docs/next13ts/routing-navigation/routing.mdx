### Resumo

O novo roteador de aplicativos em Next 13.5 usa convenção em vez de configuração para definir rotas. Ele procura arquivos especiais como `page.tsx`, `layout.tsx`, `loading.tsx`, `route.tsx`, etc.

#### **Como crio uma nova rota?**

Crie uma pasta dentro da pasta `app`.   Em cada pasta dentro da pasta app, o Next irá procurar pelos arquivos especiais `page.tsx`, `layout.tsx`, `loading.tsx`, `route.tsx` para criar a rota. Dentro dessa pasta, outros arquivos de suporte (componentes, serviços, etc) não serão considerados rotas.

**Vantagem**. Isso ajuda a organizar melhor os projetos, porque podemos manter arquivos altamente relacionados próximos uns dos outros. Não há necessidade de despejar todos os componentes em uma pasta de componentes centralizada.

<pre>

└── app
           └── pastaDaNovaRota
                      └── page.js       // <só este arquivo virará uma rota>
			         ├── componente.tsx

<pre>

#### Rotas dinâmica

Uma *rota dinâmica* é aquela que utiliza um ou mais parâmetros. Para adicionar parâmetros às nossas rotas, colocamos os nomes das pastas entre colchetes (por exemplo, `[id]`).

Em aplicações React padrão, usamos o `state hook`  para gerenciar o estado do componente. Em aplicativos renderizados no servidor, entretanto, usamos parâmetros de *query string* para manter o estado. Isso também nos permite dar *bookmark* em  páginas com estados específicos. Por exemplo, podemos dar `bookmark` em uma lista de produtos filtrada e ordenada.

Usamos arquivos de *layout* (`layout.tsx`) para criar UI que é compartilhada entre múltiplas páginas. O *layout* raiz (`/app/layout.tsx`) define a UI comum para todas as nossas páginas. Podemos criar *layouts* adicionais para áreas específicas de nossa aplicação (por exemplo, `/app/admin/layout.tsx`).

Para proporcionar uma navegação suave entre as páginas, o componente `Link` busca previamente os *links* que estão na janela de visualização.

**Cache**. À medida que o usuário se movimenta pela aplicação, o Next.js armazena o conteúdo da página em um cache no cliente. Portanto, se eles revisitarem uma página que já existe no cache, o Next.js simplesmente a captura do cache em vez de fazer uma nova solicitação ao servidor. O cache do cliente existe na memória do navegador e dura uma sessão inteira. Ele é redefinido quando fazemos uma atualização completa.
